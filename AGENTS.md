# AGENTS.md

このファイルは「開発ルールの単一ソース」です。迷ったらこの内容を優先します。

## 1. 目的

- 実装判断を迷わないようにする
- ルールを都度改善して、プロジェクトと一緒に育てる

## 2. 開発ルール（必須）

- すべての機能は `TDD` で進める（`Red -> Green -> Refactor`）
- 1PR 1目的を守る（複数機能を混ぜない）
- 外部I/Oは `src/infra`、純粋ロジックは `src/domain` に分離する
- 初期開発では `sample-content/` を配信リソースの基準として使う
- 仕様が曖昧な場合は先に `docs/decision-log.md` に判断を書いてから実装する
- 受け入れ基準に紐づかない実装は入れない

## 3. ルール更新の運用

- ルール変更は PR で行う
- 変更時は理由を1行で記録する
- 破壊的変更（大きな方針転換）は `docs/decision-log.md` に残す
