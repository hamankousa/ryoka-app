# UI/UXコンセプト（Keiteki Flow）

最終更新: 2026-02-19

## 1. コンセプト

- キーワード: `迷わない` `読みやすい` `移動が滑らか`
- 目的:
  - 曲の発見から再生・閲覧・保存までを最短で完了できること
  - 画面遷移時の認知負荷を下げること
  - 全画面で同じ操作感を維持すること

## 2. 体験原則

1. ナビゲーションは常に明示する
- 非タブ画面は必ず `戻る` を表示する
- ジェスチャー依存にしない（Android/Webでも戻れる）

2. 画面遷移は短く一貫させる
- 画面遷移は `200-260ms` の短いモーションに統一する
- 主要画面は `フェード + 微小スライド` で表示する

3. 可読性を最優先する
- 歌詞/楽譜/曲情報は装飾より情報密度と視認性を優先する
- 余白、カード角丸、境界線のルールを揃える

4. 操作結果を即時に伝える
- DLや再生など状態変化はラベルとボタン状態で明示する
- Webで無効な操作は理由を表示する

## 3. 実装方針

- ナビゲーション:
  - ルートStackに遷移アニメーションを設定
  - タブ外遷移画面に統一バックボタンを適用
- モーション:
  - 画面表示の共通エントランスアニメーションを導入
- ビジュアル:
  - 背景アクセント、カード、チップ、CTAのトーンを統一
  - ミニプレイヤー/タブバーと干渉しない余白を共通化
